# TikTok 东南亚印花T恤 AI生成系统 — 设计文档

> 日期：2026-06-04 | 状态：已确认

---

## 1. 项目概述

### 1.1 背景

TikTok 东南亚运营团队需要大量上架印花 T 恤商品。目前有大量采集到的印花 T 恤产品图，希望通过 AI 学习后裂变生成新印花，并自动生成对应的商品标题、描述等信息，最终上架到 TikTok Shop 的印尼和泰国店铺。

### 1.2 核心功能

1. **印花裂变生成** — 上传原始印花图，AI 学习后生成多个风格相似但不同的新印花
2. **商品信息生成** — AI 根据印花自动生成商品标题、描述、卖点、尺码（泰文 or 印尼文）
3. **抠图与效果图** — 自动去除背景，将印花贴到 T 恤 Mockup 上生成效果图
4. **T恤模板管理** — 上传自己的 T 恤底图（白/黑/彩色），生成时选择模板
5. **产品管理与导出** — 按国家筛选产品，分店铺导出 CSV + 图片包

### 1.3 规模

- 日处理：10000 张
- 用户：2-5 人小团队
- 目标市场：印尼 + 泰国（后续可扩展）

                 │ Product  │     │ 标题(对应语言)   │
                 ├─────────┤     │ 描述(对应语言)   │
                 │ 新印花图  │     │ 尺码             │
                 │ 效果图    │     │ 所属国家         │
                 │ 标题     │     │ 原始印花          │

### 3.2 关键关系

- 一个国家下有 N 家店铺，每家店铺绑定一个负责人
- 产品直接归属到国家，不绑定具体店铺
- T 恤模板由用户自行上传（白/黑/彩色），生成产品时选用
- 导出时按国家筛选产品，再按店铺分配上架
- 同一张原始印花可裂变给不同国家，各自生成对应语言版本

## 4. 任务流程

### 4.1 主流程

```
用户上传原始印花图
        │
        ▼
┌──────────────────┐
│ 1. 预处理          │  Celery任务
│  抠图(如有模特)     │  自动检测图片类型
│  裁剪+去背景       │  透明底输出
└──────┬───────────┘
       │
       ▼
┌──────────────────┐
│ 2. AI印花生成      │  ComfyUI (SDXL)
│  风格分析          │  img2img / ControlNet
│  生成N个变体       │  每批4张
│  T恤Mockup效果    │  IP-Adapter + 用户模板
└──────┬───────────┘
       │
       ▼
┌──────────────────┐
│ 3. AI内容生成      │  Ollama (Qwen2.5)
│  印花特征分析      │
│  生成标题+描述     │  泰文 or 印尼文
│  生成尺码信息      │
└──────┬───────────┘
       │
       ▼
┌──────────────────┐
│ 4. 结果写入        │
│  图片→ MinIO      │
│  信息→ PostgreSQL │
│  产品状态→已完成   │
└──────┬───────────┘
       │
       ▼
┌──────────────────┐
│ 5. 导出            │
│  按国家筛选        │
│  分配店铺          │
│  导出CSV+图片包    │
└──────────────────┘
```

### 4.2 设计要点

- **批量上传** — 一次拖入多张印花，自动拆成独立任务并行处理
- **步骤1可跳过** — 干净印花图直接进入生成
- **生成数量可配** — 默认每张原图裂变 4 张，设置中可调
- **语言自动匹配** — 根据目标国家字段自动输出泰文或印尼文
- **失败重试** — 每步自动重试 3 次，最终失败标记并通知

## 5. AI 引擎设计

### 5.1 自建推理架构

```
┌─────────────────────────────────────────┐
│          RTX 5070 推理服务器              │
│                                         │
│  ┌─────────────┐  ┌──────────────┐      │
│  │ ComfyUI     │  │ Ollama       │      │
│  │ (图像生成)   │  │ (图像分析+文本) │      │
│  └──────┬──────┘  └──────┬───────┘      │
│         │                │              │
│         └──────┬─────────┘              │
│                │  HTTP API              │
└────────────────┼────────────────────────┘
                 │
        ┌────────┴────────┐
        │  Django/Celery  │
        │  调度+队列        │
        └─────────────────┘
```

### 5.2 图像生成

| 配置项 | 选型 | 说明 |
|--------|------|------|
| 推理框架 | ComfyUI | 工作流可视化，API 成熟 |
| 基础模型 | SDXL / SD 3.5 Medium | 1024×1024 高质量 |
| 印花 LoRA | 训练 3-5 个风格 LoRA | 几何/花卉/卡通/文字/复古 |
| T恤 Mockup | ControlNet + IP-Adapter | 印花贴到用户上传的T恤模板上 |

### 5.3 文本生成

| 配置项 | 选型 | 说明 |
|--------|------|------|
| 推理框架 | Ollama | 一行起服务，管理方便 |
| 模型 | Qwen 2.5 / Llama 4 | 支持泰文+印尼文+中文 |
| 量化 | Q4/Q5 | 留显存给图像生成 |

### 5.4 显存调度（12GB）

| 进程 | 显存占用 | 备注 |
|------|---------|------|
| SDXL（图像） | 6-8GB | 仅生成时占用 |
| Ollama（文本） | 3-4GB | 常驻，Q4 量化 |
| 余量 | 0-3GB | |
| **策略** | 图像和文本**错峰运行** | Celery 控制，不抢显存 |

### 5.5 变体策略

| 变体方向 | 做法 | 说明 |
|---------|------|------|
| 换色系 | 保持构图，改色调 prompt | 同一款印花出多色版本 |
| 风格迁移 | 改风格关键词 | 卡通→水彩→街头→极简 |
| 元素加减 | 加 prompt 细节 | 加花/加动物/加文字 |
| 构图微调 | 调高 img2img strength | 保留风格感，换布局 |

### 5.6 代码抽象

```python
class AIProvider:
    def generate_image(self, prompt, image, params) -> ImageResult
    def analyze_image(self, image) -> AnalysisResult
    def generate_text(self, prompt) -> TextResult

class ComfyUIProvider(AIProvider):   # 自建
    ...

class ReplicateProvider(AIProvider):  # 备用
    ...
```

配置文件中切换一行即可迁移，不影响业务代码。

## 8. 错误处理

### 8.1 失败重试策略

| 任务步骤 | 重试次数 | 重试间隔 | 最终失败处理 |
|---------|---------|---------|------------|
| 预处理 | 3 | 10s | 标记失败，保留原图 |
| 图像生成 | 2 | 30s | 标记失败，不生成文本 |
| 文本生成 | 3 | 5s | 标记「待补全」，图片已生成 |
| 写入存储 | 5 | 5s | 告警 |

### 8.2 异常场景

| 场景 | 处理 |
|------|------|
| ComfyUI 无响应 | 任务自动暂停，队列等待，通知管理员 |
| Ollama 超时 | 切换到备用模型，或标记待重试 |
| 磁盘空间不足 | 触发告警，自动清理超过 30 天的缩略图 |
| GPU 显存溢出 | Celery 任务错峰调度，失败任务降级参数重试 |
